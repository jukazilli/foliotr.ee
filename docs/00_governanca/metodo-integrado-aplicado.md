## Objetivo do regime

Consolidar a documentacao do projeto em uma estrutura governavel, alinhada ao codigo real e ao Metodo Estrutural Integrado.

## Regras

- Codigo real prevalece sobre documento antigo.
- Documento ativo prevalece sobre legado.
- Prototipo nao cria requisito sozinho.
- Lacuna relevante vira backlog antes de virar implementacao.
- Slice so fecha com evidencia objetiva.
- O projeto nao deve ser reescrito inteiro em uma unica entrega documental.
